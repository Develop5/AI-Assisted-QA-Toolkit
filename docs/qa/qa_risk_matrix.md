# QA Risk Matrix

## 1. Risk Levels
- **High Risk** — critical functionality, high user impact
- **Medium Risk** — important functionality, moderate impact
- **Low Risk** — minor functionality, low impact

## 2. Risk Factors
- Business impact
- Failure history
- Component complexity
- User frequency
- Integration depth

## 3. Risk-Based Testing Strategy
- High risk → full regression + automation
- Medium risk → partial regression + smoke tests
- Low risk → basic validation only

## 4. Review Cycle
Risk matrix must be updated:
- Every quarter
- After major releases
- After critical incidents
