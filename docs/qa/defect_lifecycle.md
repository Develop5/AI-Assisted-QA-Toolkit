# Defect Lifecycle

## 1. Stages
1. **New** — defect reported
2. **Triaged** — severity and priority assigned
3. **Assigned** — developer responsible
4. **In Progress** — fix underway
5. **Ready for QA** — fix deployed to test environment
6. **Verified** — QA confirms fix
7. **Closed** — defect resolved
8. **Reopened** — fix failed or regression detected

## 2. Severity Levels
- Critical: system unusable
- High: major functionality broken
- Medium: partial functionality broken
- Low: cosmetic or minor issue

## 3. Priority Levels
- P1: fix immediately
- P2: fix in next sprint
- P3: fix when possible

## 4. Mandatory Fields
- Title
- Steps to reproduce
- Expected result
- Actual result
- Environment
- Logs or screenshots

## 5. QA Responsibilities
- Validate defects
- Provide clear reproduction steps
- Verify fixes
- Detect regressions
